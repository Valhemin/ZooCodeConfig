# DESIGN.md System

Use for UI-heavy work or new web projects.

DESIGN.md should capture:
- Brand/product personality.
- Layout principles.
- Typography scale.
- Color tokens.
- Spacing and radius tokens.
- Component states.
- Accessibility rules.
- Anti-patterns to avoid.

Implementation should follow DESIGN.md or update it when the design system changes.
