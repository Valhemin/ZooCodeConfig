name: spec-consistency-analysis
description: Run read-only cross-artifact analysis across spec, plan, tasks, checklists, and constitution.
---

Analyze cross-artifact consistency before implementation.

Build internal maps:
- requirements inventory
- user stories / acceptance criteria
- task coverage mapping
- constitution/principle rules
- data/API/UI entities and naming

Report only high-signal issues with severity. Do not modify files during analysis.

Prioritize constitution violations, uncovered baseline requirements, conflicting requirements, untestable criteria, and task order contradictions.
