---
name: writing-plans-superpowers
description: Write implementation plans that are concrete, small, verifiable, and low-ambiguity.
---

Use after a design/spec is accepted.

A good plan includes:
- exact files or file areas
- ordered tasks
- dependencies
- acceptance criteria
- verification steps
- rollback notes for risky changes
- clear boundaries for what is out of scope

Tasks should be small enough to review independently and concrete enough that another agent can execute them without guessing.
