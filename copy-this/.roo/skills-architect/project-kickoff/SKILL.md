# Project Kickoff

## When to Use

- Starting a brand-new project or product.
- Starting a large feature with no existing spec.
- Architect Pro is invoked and no spec, task list, or plan exists yet.
- User says: "I want to build [X]", "Start a new project for [X]", "We're kicking off [X]".

Do NOT use for continuing in-progress work — use `lead-decomposition` or `delivery-roadmap` instead.

## Lead Output

- Goal and product framing.
- Critical unknowns.
- Suggested tech direction.
- Required spec pack.
- Delivery milestones.
- Subtask delegation plan (what, to whom, in what order, artifact destination).
- Production-readiness gates.

## Delegation Step

Delegate Architect Pro first unless the user already has complete specs.

State explicitly before delegating:
- What Architect Pro will produce.
- Where it lands (e.g., `spec/`, `docs/`).
- What Architect Pro will do with the output (synthesize into milestone plan).

## After Architect Pro Returns

Architect Pro produces a synthesis (see coordination contract): spec coverage, gaps, risks, and the concrete next milestone with delegation plan. Does not relay Architect Pro output raw.
