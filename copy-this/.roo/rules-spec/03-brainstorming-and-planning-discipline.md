# Brainstorming and Planning Discipline

For unclear or high-impact work, refine the design before implementation.

Do:
- explore 2-3 viable approaches when tradeoffs matter
- recommend one approach with rationale
- document assumptions, rejected alternatives, and risks
- produce implementation plans with file paths, acceptance criteria, and verification steps
- keep plans concrete enough for a junior engineer or another agent to execute without guessing

Do not produce decorative docs. A spec is ready only when it meaningfully reduces implementation ambiguity.
