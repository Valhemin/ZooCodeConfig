# Production Readiness Gate

Before claiming completion, verify as much as practical.

Use `rules/05-definition-of-done.md` as the task-type checklist (API, UI, all tasks). This gate governs the final status output format; the DoD governs what must be checked before reaching this gate.

Final status must include:
- What was completed.
- What files/areas changed.
- Verification commands run and results.
- What was not run and why.
- Known risks, follow-ups, or manual QA needed.
- Deferred DoD items (if any) with reason.

Do not call work production-ready if critical acceptance criteria are incomplete, tests fail, or verification was skipped without a clear reason. See `rules/05-definition-of-done.md` for the full 95%+ production-ready definition.
