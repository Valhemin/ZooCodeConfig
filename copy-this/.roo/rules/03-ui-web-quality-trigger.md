# UI and Web Quality Trigger

When a task touches frontend UI, product UX, landing pages, dashboards, forms, components, or visual polish:

- Inspect existing components, tokens, DESIGN.md, styles, layout conventions, and assets before designing.
- Avoid generic AI-looking UI patterns unless the existing product intentionally uses them.
- Consider information hierarchy, spacing rhythm, typography, states, responsive behavior, accessibility, empty/loading/error states, and microcopy.
- Prefer updating or creating DESIGN.md for UI-heavy projects.
- Use Playwright or screenshot-based QA when available for visual work.
