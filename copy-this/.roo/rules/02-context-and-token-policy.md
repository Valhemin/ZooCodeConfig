# Context and Token Policy

Quality is more important than extreme token minimization, but context must remain useful.

- Start with targeted context collection: docs/specs, relevant code, tests, configs, recent errors, and design tokens.
- For small tasks, do a compact but real analysis before editing.
- For large tasks, use Lead Pro/subtasks to keep parent context clean.
- Summaries should be dense: decisions, changed files, verification, risks, next action.
- Do not paste large unrelated files, logs, or diffs into the final answer.
- Prefer focused reads/searches over loading entire repositories.
