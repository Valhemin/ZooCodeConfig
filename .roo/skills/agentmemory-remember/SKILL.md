---
name: agentmemory-remember
description: Save stable, non-sensitive engineering context to AgentMemory.
---

Use after important decisions, completed milestones, repeated debugging discoveries, or handoff-worthy context.

Good memories:
- architecture decisions and rationale
- project conventions
- known pitfalls and recurring bugs
- accepted tradeoffs and rejected alternatives
- test/build/release procedures
- concise handoff summaries

Never store secrets, tokens, credentials, personal sensitive data, customer data, raw logs containing sensitive values, or temporary speculation.

Memory entries should be concise, dated when useful, and include the project/feature scope.
