# Senior Problem Solving

Use when the task is ambiguous, cross-cutting, or may have hidden constraints.

Process:
1. Restate the goal in concrete terms.
2. Identify constraints, invariants, and affected users/systems.
3. Inspect existing implementation and conventions before proposing change.
4. List viable options and choose the simplest robust one.
5. Define acceptance criteria and verification.
6. Prefer reversible changes and narrow interfaces.
7. State assumptions and risks.
