---
name: verification-before-completion
description: Require evidence before claiming completion.
---

Use before final summaries, task completion, release readiness, or bug fix claims.

Completion requires:
- acceptance criteria checked
- relevant tests/build/lint/typecheck/manual QA run or explicitly skipped with reason
- known failures reported honestly
- changed files summarized
- risks and follow-ups listed

Never write "done", "fixed", or "production-ready" without evidence.
