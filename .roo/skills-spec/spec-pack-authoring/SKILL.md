# Spec Pack Authoring

Use for new projects and major features.

Create a spec pack that is useful for implementation, not just impressive:
- Brief and goals.
- Product requirements.
- UX/UI spec.
- Architecture.
- Data model.
- API contract.
- Implementation plan.
- Test plan.
- Risks and edge cases.
- Release checklist.

Every spec must include assumptions, unknowns, acceptance criteria, and verification.
